# AI-FDE 完整技术栈 + 硅谷大厂高频面试考点
## 一、FDE 分层技术栈（按求职权重排序）
### 1. 核心AI应用层（面试重中之重，占技术面60%）
#### 大模型基础
- LLM底层：Tokenization、上下文窗口、温度/topP、幻觉成因、上下文溢出、流式输出
- 厂商API：OpenAI、Anthropic Claude、Google Gemini、本地推理vLLM/TGI/TensorRT-LLM
- Prompt工程：系统提示词分层、Few-shot、思维链CoT、结构化JSON输出、Prompt版本管理、回归评测

#### RAG 全链路（FDE最常落地业务）
- 向量化：sentence-transformers、Embedding选型、重排Rerank、文本分块策略（递归/语义分块）
- 向量库：FAISS、Chroma、Milvus、Pinecone、Weaviate
- 检索优化：混合检索、关键词+向量、过滤元数据、去重、召回精度调优
- 生产问题：文档乱码、长文档截断、私有数据隐私、检索噪声抑制

#### Agent 智能体（OpenAI/Anthropic必考）
- 框架：LangChain、LlamaIndex、AutoGen、OpenAI Agents SDK
- 核心机制：Function Calling、Tool调用、ReAct规划、多轮状态持久化、Agent自省纠错
- 生产约束：工具权限管控、循环熔断、任务超时、成本限流

#### LLM观测与评测LLMOps
- 追踪：LangSmith、Braintrust、LangFuse
- 评测指标：RAG召回准确率、答案忠实度、幻觉检测、任务完成率、人工标注基准
- 安全风控：数据脱敏、输入输出过滤、权限隔离、审计日志

### 2. 后端工程层（生产交付门槛）
- 主力语言：**Python（必精通）**，辅助TS/Go/Java
- Web服务：FastAPI、Flask，接口限流、重试、熔断、鉴权、流式SSE
- 数据存储
  - 关系库：PostgreSQL、MySQL（复杂JOIN、窗口函数、数据建模）
  - 缓存：Redis（向量缓存、会话存储、限流器）
  - 消息队列：Kafka/RabbitMQ（异步文档处理、Agent任务队列）
- ETL数据处理：PDF/Word/Excel解析、脏数据清洗、结构化数据抽取

### 3. 云原生&私有化部署（驻场客户核心技能）
- 容器：Docker、Docker Compose、K8s基础（Pod/Service/ConfigMap）
- CI/CD：GitHub Actions、ArgoCD、一键部署脚本
- 云厂商：AWS/GCP/Azure（硅谷主流）、国内阿里云/华为云
- 私有化适配：离线隔离环境、混合云、内网代理、客户老旧系统兼容
- 可观测：日志ELK、指标Prometheus、告警链路追踪

### 4. 企业集成&安全合规（高薪区分点）
- 权限体系：RBAC、SSO/OAuth2、企业统一登录
- 系统对接：CRM/ERP/OA、API网关、第三方业务系统API适配
- 数据合规：数据脱敏、本地数据不出域、行业监管（金融/医疗）
- 网络：内网穿透、防火墙、API访问白名单

### 5. 辅助全栈工具
- 前端基础：React/Next简单Demo页面（快速给客户演示）
- 脚本：Bash、自动化运维脚本
- AI辅助开发：Cursor、Claude Code、Copilot


## 二、硅谷FDE完整面试高频考点（OpenAI/Anthropic/Palantir通用）
### 模块1：算法编码（基础轮，中等难度，偏工程）
1. 基础算法：数组、哈希、滑动窗口、二分、DFS/BFS，不考极难竞赛题
2. 高频实操编程题
   - 实现LLM流式SSE接口，处理断连重试
   - 文档分块工具：递归文本分割、过滤无用段落
   - 向量检索去重、混合检索逻辑实现
   - 通用API请求封装：限流、指数退避、异常捕获
   - SQL：多表业务数据统计、用户会话数据分析
3. 考察重点：**工程健壮性**（空值、异常、边界、日志、测试用例），不追求极致算法优化

### 模块2：AI系统设计（核心技术面，占比最高）
#### RAG 系统设计（必考题）
1. 为银行搭建百万合同知识库RAG，如何保证无幻觉、检索精准、低成本？
2. RAG效果差怎么排查：召回低、答案偏离、上下文冲突优化方案
3. 对比：微调 vs RAG vs 超长Prompt，不同业务场景取舍（成本/数据量/延迟）
4. 多模态RAG：PDF图表、扫描文档如何处理

#### Agent 生产系统设计
1. 设计企业工单自动处理Agent，包含工具调用、失败重试、人工介入机制
2. Agent无限循环、错误调用工具如何熔断与监控
3. 多Agent协作架构设计，任务分发、状态持久化

#### 私有化大模型部署设计
1. 客户内网离线环境部署LLM+RAG完整方案，无公网API
2. 多租户AI平台权限、数据隔离架构
3. 高峰期并发限流、成本控制、推理延迟优化

#### LLM可观测与评测
1. 设计一套自动化评测流水线，监控Prompt迭代后效果退化
2. 如何自动检测模型输出幻觉，落地告警机制

### 模块3：客户业务Case Study（FDE独有核心轮，高薪分水岭）
全部模拟**驻场客户沟通全流程**，考察业务拆解、需求收敛、项目落地闭环
1. 制造工厂：想用AI分析产线日志减少故障，客户只有模糊需求，90天交付路线
2. 保险企业：百万理赔单自动摘要、风险审核，合规严格、数据敏感
3. 投行：私有财报智能问答，禁止数据外传，高并发查询需求
4. 通用提问模板：
   - 客户只说“我们需要AI提升效率”，你如何挖掘真实痛点、量化ROI？
   - 客户提出不切实际需求（实时千亿文档查询、零延迟Agent），如何沟通折中方案？
   - 项目交付后效果不达预期，现场如何快速迭代修复、说服客户验收？
   - 客户内部多部门需求冲突，如何协调优先级

### 模块4：工程运维&部署面试题
1. 一套AI服务客户现场频繁崩溃，如何完整排障链路（日志、监控、资源、API限流）
2. 客户老旧Windows内网系统，无法上云，如何轻量化部署RAG应用
3. CI/CD流水线设计，自动化测试Prompt与Agent回归用例
4. 大模型API成本暴涨，设计监控与节流方案

### 模块5：行为/业务综合面试（Hiring Manager终面）
1. 过往完整AI落地项目复盘：从需求调研→PoC→交付→迭代完整讲清，突出**客户价值、解决的复杂现场问题**
2. 你理解的FDE和普通后端/算法工程师区别是什么？
3. 长期驻场客户、频繁出差、处理混乱业务系统，你如何平衡技术与业务？
4. 技术方案和客户业务目标冲突，你怎么决策？
5. AI安全、幻觉、数据隐私在企业落地中如何平衡功能与风险（Anthropic必考）
6. 如何沉淀客户通用需求，反哺公司产品标准化组件？

### 模块6：行业对比高频概念问答（口述简答题）
1. Token窗口溢出解决方案
2. Embedding维度、向量库索引选型差异（FAISS vs Milvus）
3. Function Calling失败常见原因与优化
4. 幻觉产生根源、生产级抑制手段
5. 私有数据微调数据清洗标准
6. 流式输出和一次性返回的适用场景
7. 混合云/离线私有化部署优缺点
8. Prompt A/B测试落地流程
